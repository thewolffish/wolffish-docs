# التهيئة للنجاح

صُمم Wolffish لاكتشاف الأمور. يستطيع تثبيت الأدوات المفقودة، والتنقل في واجهات غير مألوفة، والتعافي من الأخطاء، والتكيف مع أي حالة يجد عليها جهازك. على جهاز عادي، يتعامل مع معظم الإعداد تلقائياً.

لكن "يستطيع التعامل" و"يجب أن يتعامل" أمران مختلفان. كل خطوة إضافية يتخذها الوكيل تكلف رموزاً، وتحرق وقتاً، وتضيف نقطة فشل. عندما يُطلق heartbeat في الساعة 9 صباحاً ويقضي الوكيل 4 دقائق في تثبيت Node.js، و5 دقائق في إعادة محاولة أذونات لقطة الشاشة التي لا يملكها، ودقيقتين في التنقل إلى صفحة كان بإمكانك تركها مفتوحة — فهذا 11 دقيقة من العمل الضائع قبل أن تبدأ المهمة الفعلية حتى.

النموذج الذهني: **هيئ المكتب قبل أن يجلس الوكيل.** امنح الأذونات مسبقاً. ثبّت الأدوات مسبقاً. اترك التطبيقات الصحيحة مفتوحة. كلما قل ما يحتاج الوكيل لاكتشافه، كانت التشغيلة أسرع وأرخص وأكثر حتمية.

تغطي هذه الصفحة المتطلبات والأذونات العامة. كل حالة استخدام تسرد أيضاً إعدادها المحدد الخاص بها — لكن ما هنا ينطبق على جميعها.

## أذونات macOS

يعمل Wolffish كتطبيق macOS. للتحكم بسطح مكتبك — التقاط لقطات الشاشة، وتحريك الفأرة، والكتابة، وإرسال أوامر إلى تطبيقات أخرى — يحتاج إلى إذن صريح من macOS. هذه ليست إعدادات Wolffish؛ إنها بوابات على مستوى نظام التشغيل في **إعدادات النظام > الخصوصية والأمان**.

إذا كان أي منها مفقوداً، لا يحصل الوكيل على خطأ واضح. يحصل على إخفاقات صامتة، ولقطات شاشة فارغة، واستجابات "غير مسموح به" يعيد محاولتها مراراً وتكراراً، مستهلكاً الرموز والوقت.

<Accordion title="تسجيل الشاشة">
**ما تفعله:** تتيح لـ Wolffish التقاط لقطات شاشة لسطح مكتبك. مطلوبة من كل قدرة تستخدم `computer-use`.

**ما يحدث بدونها:** يُعيد `computer_screenshot` بصمت لا شيء — لا خطأ، فقط نتيجة فارغة. يُعيد الوكيل المحاولة حتى 10 مرات، مستهلكاً ~300 ثانية لكل محاولة قبل الاستسلام. ليس لديه طريقة لمعرفة أن الإذن مفقود.

**كيفية منحها:**
1. افتح **إعدادات النظام > الخصوصية والأمان > تسجيل الشاشة**
2. انقر على زر `+`، ابحث عن **Wolffish** وأضفه
3. **أعد تشغيل Wolffish** — يتطلب هذا الإذن إعادة تشغيل التطبيق ليسري
</Accordion>

<Accordion title="إمكانية الوصول">
**ما تفعله:** تتيح لـ Wolffish التحكم بالفأرة ولوحة المفاتيح. مطلوبة من كل قدرة تستخدم `computer-use`.

**ما يحدث بدونها:** يفشل `computer_mouse_click` و`computer_keyboard_type` فوراً بـ "غير مسموح به".

**كيفية منحها:**
1. افتح **إعدادات النظام > الخصوصية والأمان > إمكانية الوصول**
2. انقر على زر `+`، ابحث عن **Wolffish** وأضفه
3. تسري فوراً — لا حاجة لإعادة التشغيل
</Accordion>

<Accordion title="الأتمتة">
**ما تفعله:** تتيح لـ Wolffish إرسال أوامر إلى تطبيقات أخرى (Chrome وFinder وTerminal وغيرها) عبر AppleScript. مطلوبة لـ `computer-use` وأي قدرة تقود تطبيقاً آخر.

**ما يحدث بدونها:** يفشل أي استدعاء `osascript` بـ "غير مصرح بإرسال أحداث Apple". يُعيد الوكيل المحاولة — في إحدى تشغيلات الاختبار استهلك ~264 ثانية على 6 محاولات قبل ظهور موجِّه macOS.

**كيفية منحها:**
1. افتح **إعدادات النظام > الخصوصية والأمان > الأتمتة**
2. ابحث عن **Wolffish** في القائمة
3. فعّل كل تطبيق هدف — كحد أدنى، **Google Chrome** و**Finder**
4. قد يطلب منك macOS أيضاً عند أول استخدام — انقر على "موافق" عند ظهور الطلب

يمكنك منحها مسبقاً لتجنب مقاطعة الموجِّه لتشغيلة heartbeat في منتصف التنفيذ.
</Accordion>

<Warning>
**امنح الأذونات الثلاثة قبل أول تشغيلة مستقلة.** إذا كان أي إذن مفقوداً، يُطلق heartbeat، ويستنزف الوكيل ميزانية رموزه بالكامل في إعادة محاولة استدعاءات أدوات فاشلة، ولا تنجز شيئاً. تسجيل الشاشة هو الأسوأ — يفشل بصمت، فلا يعرف الوكيل حتى أنه لا يستطيع رؤية الشاشة.
</Warning>

## ما تحتاجه كل قدرة

ليست كل حالة استخدام بحاجة إلى كل إذن. إليك ما تمنحه بناءً على ما يستخدمه سير عملك:

| القدرة | أذونات macOS | أدوات النظام | مفاتيح API |
|------------|-------------------|--------------|----------|
| `computer-use` | تسجيل الشاشة + إمكانية الوصول + الأتمتة | Chrome (أو المتصفح الهدف) | مزود سحابي (Anthropic أو OpenAI) |
| `web-search` | لا شيء | لا شيء | مزود سحابي. [Brave Search API](https://brave.com/search/api/) _(اختياري — يحسّن الجودة)_ |
| `shell` | لا شيء (يعمل في عملية Wolffish الخاصة) | أي أدوات CLI تحتاجها المهمة (git، ffmpeg، إلخ) | لا شيء |
| `browser` | لا شيء | Chromium لـ Playwright (~200MB، يُنزل تلقائياً عند أول استخدام) | لا شيء |
| `document`، `pdf`، `spreadsheet` | لا شيء | Node.js (لاعتماديات npm) | لا شيء |
| `github` | لا شيء | Git، Node.js | رمز وصول شخصي لـ GitHub |
| `notion` | لا شيء | Node.js | مفتاح Notion API |
| `google-workspace` | لا شيء | Node.js | OAuth جوجل (مُكوَّن في الإعدادات > التكاملات) |

**النمط:** إذا استخدم سير العمل `computer-use`، فأنت بحاجة إلى أذونات macOS الثلاثة بالإضافة إلى مزود سحابي. إذا استخدم فقط `web-search` و`shell`، فقد لا تحتاج إلى شيء سوى مفتاح API سحابي.

## أدوات النظام

يثبّت Wolffish تلقائياً معظم أدوات النظام عندما يحتاجها — Node.js، Homebrew، Chromium لـ Playwright، إلخ. على جهاز عادي ليس مُحصناً أو مُقيداً، يعمل هذا بشكل جيد.

لكن التثبيت التلقائي غير حتمي. يضيف دقائق إلى التشغيلة الأولى، يمكن أن يُطلق موجِّهات كلمة مرور توقف heartbeat، ويختار أحياناً مسار تثبيت خاطئ. التثبيت المسبق لهذه الأدوات يجعل بيئة الوكيل قابلة للتنبؤ.

### Node.js v18+

مطلوب من 8+ قدرات (browser، document، pdf، spreadsheet، web-search، github، notion، computer-use) لتثبيت اعتماديات npm الخاصة بها. npm مُجمَّع مع Node.js.

سيحاول Wolffish تثبيت Node.js إذا كان مفقوداً. لكن إذا كنت تستخدم nvm، فقد لا يكون ثنائي Node.js على PATH النظام الذي يراه Wolffish — قد يحاول الوكيل تثبيت نسخة ثانية عبر Homebrew بينما توجد نسخة سليمة تحت nvm. التحقق المسبق يتجنب هذا:

```bash
# Check if Node.js is visible on the system PATH
node -v

# If using nvm, make sure the default alias is set
nvm alias default 18
```

### Homebrew (macOS)

يُثبَّت تلقائياً بواسطة مدير حزم Wolffish إذا كان مفقوداً، لكن التثبيت يُطلق موجِّه كلمة مرور ويستغرق عدة دقائق. إذا كنت على macOS، ثبّته مسبقاً:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### Chromium لـ Playwright

تستخدم قدرة `browser` Playwright لقيادة نسخة Chromium بدون رأس. تُنزّل التشغيلة الأولى ~200MB. ثبّت مسبقاً لتخطي الانتظار:

```bash
npx playwright install chromium
```

### Git

تستخدمه قدرة `git`. مثبّت مسبقاً على معظم أنظمة macOS وLinux. تحقق بـ `git --version`.

## تقليل تكلفة الرموز ووقت التشغيل

كل تحسين أدناه يوفّر رموزاً، ويقلل وقت ساعة الحائط، ويجعل التشغيلة أكثر موثوقية. تتراكم عبر تشغيلات heartbeat اليومية.

### اترك التطبيقات الصحيحة مفتوحة

إذا استخدم سير العمل Chrome، اترك Chrome مفتوحاً على الصفحة الهدف. يتخطى الوكيل تشغيل المتصفح والتنقل وتحميل الصفحة — موفراً ~3 لقطات شاشة و~800 رمز لكل تشغيلة.

### حافظ على الجلسات نشطة

إذا احتاج سير العمل إلى حالة تسجيل دخول (LinkedIn، Gmail، GitHub في المتصفح)، تأكد أن الجلسة لم تنتهِ. جدار تسجيل دخول في منتصف التشغيلة يضيف 5+ استدعاءات أدوات لم يحسبها الوكيل.

### استخدم مزوداً سحابياً للمهام كثيفة الحكم

تعمل النماذج المحلية (Ollama) لسير عمل استدعاء أدوات بسيط، لكن أي شيء يتطلب تنسيقاً أو كتابة أو استدلالاً متعدد الخطوات يجب أن يستخدم مزوداً سحابياً. **DeepSeek V4 Pro هو الخيار الموصى به افتراضيًا** — يطابق الأداء الوكيلي الحدودي بتكلفة أقل بـ 29–34 ضعفاً من Opus 4.7 أو GPT-5.5 على الإخراج، و~11 ضعفاً على الإدخال. استخدم Claude خصوصاً لمهام computer-use أو عندما تحتاج أسلوبه الفريد في اتباع التعليمات.

### امنح الأتمتة مسبقاً لكل تطبيق هدف

كل تطبيق جديد يحاول Wolffish التحكم به عبر AppleScript يُطلق موجِّه macOS في المرة الأولى. إذا كنت تعرف أن سير العمل يقود Chrome أو Finder أو Terminal — امنح هذه مسبقاً في إعدادات النظام > الخصوصية والأمان > الأتمتة قبل التشغيلة الأولى.

<Note>
**هذا يتعلق بالحتمية، وليس بالقدرة.** يستطيع Wolffish تثبيت Node.js، وتنزيل Chromium، والتعامل مع موجِّهات الأذونات، والتنقل إلى LinkedIn من بداية باردة. مُصمم لذلك. لكن heartbeat يعمل في 45 ثانية مع كل شيء معد مسبقاً أفضل من واحد يعمل في 12 دقيقة بينما يُهيئ الوكيل بيئته الخاصة. هيئ المكتب. دع الوكيل يقوم بالعمل.
</Note>
