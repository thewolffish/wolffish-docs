# التشغيل الأول

عند تشغيل Wolffish لأول مرة، يُنشئ مساحة العمل في `~/.wolffish/workspace/` وينسخ ملفات التكوين الافتراضية من القوالب المرفقة. إليك ما يُنشأ:

## مساحة العمل

```
~/.wolffish/workspace/
└── brain/
    ├── identity/
    │   ├── soul.md          # شخصية الوكيل
    │   └── user.md          # ملفك الشخصي
    ├── prefrontal/
    │   └── agents.md        # إجراءات الوكيل وقواعد الأدوات
    ├── cerebellum/          # القدرات (shell, filesystem, git, إلخ)
    ├── hippocampus/         # الذاكرة (الحلقات، المُدمَجة، المعرفة)
    ├── motor/tasks/         # سجلات تنفيذ المهام
    ├── basalganglia/        # سجلات الملاحظات
    ├── brainstem/           # تكوين العمليات الخلفية
    ├── corpus/              # سجلات الأحداث
    ├── conversations/       # سجل الدردشة
    └── config.json          # تكوين التطبيق
```

## خطوات الإعداد الأولي

1. **كشف Ollama** — يتحقق Wolffish من تشغيل Ollama على `localhost:11434`. إن لم يجده، يعرض إرشادك خلال تثبيته — أو يمكنك تخطي هذه الخطوة واستخدام مزودي السحابة بدلاً منه.
2. **اختيار النموذج** — إن كان Ollama متوفرًا، اختر نموذجًا محليًا. يستطيع Wolffish تنزيل النماذج مباشرة إن لم تكن مثبتة.
3. **ضبط المزوّد** — اضبط مزوّدًا واحدًا على الأقل: DeepSeek (موصى به)، أو Anthropic، أو OpenAI للنماذج السحابية، أو Ollama للاستدلال المحلي. تحتاج واحدًا على الأقل لبدء المحادثة.

## ما يحدث في كل تشغيل

بعد التشغيل الأول، كل تشغيل لاحق يقوم بما يلي:

1. يقرأ `config.json` لإعدادات النموذج والمزود
2. يهيّئ وحدات الدماغ الخمس عشرة عبر ناقل الأحداث (corpus)
3. يفهرس ملفات Markdown في مساحة العمل ضمن `cortex.db` (SQLite FTS5)
4. يبدأ العمليات الخلفية (مراقب الملفات، مجدول cron)
5. يفحص صحة المزودين (Ollama، وواجهات السحابة إن كانت مُعدة)
