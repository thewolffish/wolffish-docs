# OpenRouter (مُجمّع)

```
POST https://openrouter.ai/api/v1/chat/completions
```

يستخدم تدفق SSE بتنسيق أدوات متوافق مع OpenAI. يوجّه الطلبات إلى أي نموذج من أي مزوّد عبر مفتاح API واحد.

**OpenRouter هو مُجمّع نماذج** — نقطة نهاية API واحدة تُوكّل الطلبات إلى Anthropic وOpenAI وDeepSeek وQwen وxAI وMeta وMistral وGoogle وعشرات غيرهم. مفتاح واحد، فاتورة واحدة، وصول لكل شيء.

الأفضل لـ: تجربة نماذج لم تُعدّها مباشرة، فاتورة موحّدة، الوصول لنماذج غير مدعومة أصليًا في Wolffish.

<Warning>
  **نوصي بضبط المزودين مباشرة كلما أمكن.** التكامل المباشر يمنحك زمن استجابة أقل (بدون قفزة وسيطة)، وتتبع تكاليف دقيق، وميزات خاصة بالمزوّد (التخزين المؤقت العابر في Anthropic، وFIM في DeepSeek)، وبدون هامش وسيط. يضيف OpenRouter طبقة توجيه قد تزيد زمن الاستجابة وتسبب سلوكًا غير متسق أحيانًا.

  استخدم OpenRouter عندما تريد تجربة نماذج لم تُعدّها مباشرة، أو كبديل مناسب لمزودين لا تريد إدارة مفتاح API منفصل لهم.
</Warning>

## الحصول على مفتاح API

1. اذهب إلى [openrouter.ai](https://openrouter.ai)
2. سجّل حسابًا أو سجّل الدخول
3. انتقل إلى **Keys** وأنشئ مفتاحًا جديدًا
4. الصقه في Wolffish ← الإعدادات ← النماذج ← OpenRouter

## النماذج المدعومة

يدعم OpenRouter مئات النماذج. يُطابق Wolffish حدود الإخراج مع حدود كل مزوّد أصلي لمنع أخطاء تجاوز السياق:

| النموذج (عبر OpenRouter) | الإخراج الأقصى | ملاحظات |
|--------------------------|----------------|---------|
| anthropic/claude-opus-*، anthropic/claude-sonnet-* | 32K | يطابق حدود Anthropic الأصلية. |
| openai/gpt-5* | 64K | يطابق حدود OpenAI الأصلية. |
| openai/gpt-4.1* | 32K | الجيل السابق من OpenAI. |
| openai/gpt-4o* | 16K | مُحسّن للدردشة. |
| openai/o3*، openai/o4* | 64K | نماذج استدلال. |
| deepseek/* | 32K | يطابق حدود DeepSeek الأصلية. |
| x-ai/grok-* | 32K | يطابق حدود xAI الأصلية. |
| google/gemini-* | 64K | نماذج Google Gemini. |
| meta-llama/* | 16K | نماذج Meta Llama. |
| qwen/* | 32K | يطابق حدود Qwen الأصلية. |
| mistralai/* | 32K | نماذج Mistral. |

<Note>
  حدود الإخراج مضبوطة بشكل محافظ لضمان أن النموذج لديه دائمًا ميزانية سياق كافية لرموز الإدخال. تُطابق هذه الحدود تلك المُستخدمة في تكاملات المزودين الأصلية — تحصل على نفس السلوك سواء وصلت للنموذج مباشرة أو عبر OpenRouter.
</Note>

## التكامل المباشر مقابل OpenRouter

| | مزوّد مباشر | OpenRouter |
|--|-------------|-----------|
| **زمن الاستجابة** | الأقل — استدعاء API مباشر | أعلى — قفزة وسيطة إضافية |
| **التكلفة** | أسعار المزوّد فقط | أسعار المزوّد + هامش OpenRouter |
| **الميزات** | ميزات المزوّد الكاملة | مجموعة فرعية مُوحّدة |
| **الفوترة** | منفصلة لكل مزوّد | فاتورة موحّدة واحدة |
| **الوصول للنماذج** | المزودون المُعدّون فقط | مئات النماذج بمفتاح واحد |
| **التخزين المؤقت** | أصلي للمزوّد (Anthropic العابر، DeepSeek، إلخ) | يختلف حسب المزوّد الأساسي |

### متى تستخدم OpenRouter

**مناسب:**
- تجربة نماذج من مزودين لم تُعدّهم بعد
- اختبار A/B سريع بين عائلات نماذج مختلفة
- فاتورة موحّدة عندما تريد فاتورة API واحدة فقط
- الوصول لنماذج متخصصة أو أحدث غير مدعومة أصليًا بعد

**استخدم التكامل المباشر بدلاً من ذلك عندما:**
- المزوّد مدعوم أصليًا بالفعل (DeepSeek، Anthropic، OpenAI، إلخ)
- تحتاج أقل زمن استجابة ممكن
- تريد ميزات خاصة بالمزوّد (تخزين مؤقت، بادئات الموجّهات، إلخ)
- تُشغّل أحمال عمل إنتاجية عالية الحجم حيث تتراكم القفزة الوسيطة

<Tip>
  إذا كنت تستخدم بالفعل DeepSeek أو Anthropic أو أي مزوّد مدعوم أصليًا، أبقِ على الاتصال المباشر. أضف OpenRouter فقط لنماذج لا يمكنك الوصول إليها مباشرة. تدرج Wolffish يتعامل مع الانتقال تلقائيًا — المزودون المباشرون يُجرَّبون أولًا، ثم OpenRouter.
</Tip>

## نماذج الاستدلال عبر OpenRouter

يدعم OpenRouter نماذج الاستدلال من مزودين متعددين. يكشف Wolffish تلقائيًا نماذج الاستدلال ويُرسل المعاملات المناسبة:

- **سلسلة o من OpenAI وGPT-5**: معامل `reasoning_effort`
- **Claude Opus من Anthropic**: كائن `reasoning` بمستوى الجهد
- **سلسلة R من DeepSeek**: معامل `reasoning_effort`

وضع الاستدلال (بدون، عالي، أقصى) الذي تضبطه في واجهة Wolffish يُترجم إلى التنسيق الصحيح لكل مزوّد تلقائيًا.

## تدرج المزودين

يقع OpenRouter قرب نهاية تدرج Wolffish: **DeepSeek → MiMo → Kimi → MiniMax → xAI → Qwen → Stepfun → Claude → OpenAI → OpenRouter → Ollama**. هذا يعني أن اتصالات المزودين المباشرة تُجرَّب دائمًا أولًا. يعمل OpenRouter كبديل واسع — إذا كانت مزوداتك الأساسية والثانوية معطّلة، يمكن لـ OpenRouter التوجيه إلى نموذج بديل تلقائيًا.
