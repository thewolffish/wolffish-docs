# OpenRouter (مُجمّع)

```
POST https://openrouter.ai/api/v1/chat/completions
```

يستخدم تدفق SSE بتنسيق أدوات متوافق مع OpenAI. يوجّه الطلبات إلى أي نموذج من أي مزوّد عبر مفتاح API واحد.

**OpenRouter هو مُجمّع نماذج** — نقطة نهاية API واحدة تُوكّل الطلبات إلى Anthropic وOpenAI وDeepSeek وQwen وxAI وMeta وMistral وGoogle وعشرات غيرهم. مفتاح واحد، فاتورة واحدة، وصول لكل شيء.

الأفضل لـ: تجربة نماذج لم تُعدّها مباشرة، فاتورة موحّدة، الوصول لنماذج غير مدعومة أصليًا في وولف فيش.

<Warning>
  **نوصي بضبط المزودين مباشرة كلما أمكن.** التكامل المباشر يمنحك زمن استجابة أقل (بدون قفزة وسيطة)، وتتبع تكاليف دقيق، وميزات خاصة بالمزوّد (التخزين المؤقت العابر في Anthropic، وFIM في DeepSeek)، وبدون هامش وسيط. يضيف OpenRouter طبقة توجيه قد تزيد زمن الاستجابة وتسبب سلوكًا غير متسق أحيانًا.

  استخدم OpenRouter عندما تريد تجربة نماذج لم تُعدّها مباشرة، أو كبديل مناسب لمزودين لا تريد إدارة مفتاح API منفصل لهم.
</Warning>

## الحصول على مفتاح API

1. اذهب إلى [openrouter.ai](https://openrouter.ai)
2. سجّل حسابًا أو سجّل الدخول
3. انتقل إلى **Keys** وأنشئ مفتاحًا جديدًا
4. الصقه في وولف فيش ← الإعدادات ← النماذج ← OpenRouter

## التكامل المباشر مقابل OpenRouter

| | مزوّد مباشر | OpenRouter |
|--|-------------|-----------|
| **زمن الاستجابة** | الأقل — استدعاء API مباشر | أعلى — قفزة وسيطة إضافية |
| **التكلفة** | أسعار المزوّد فقط | أسعار المزوّد + هامش OpenRouter |
| **الميزات** | ميزات المزوّد الكاملة | مجموعة فرعية مُوحّدة |
| **الفوترة** | منفصلة لكل مزوّد | فاتورة موحّدة واحدة |
| **الوصول للنماذج** | المزودون المُعدّون فقط | مئات النماذج بمفتاح واحد |
| **التخزين المؤقت** | أصلي للمزوّد (Anthropic العابر، DeepSeek، إلخ) | يختلف حسب المزوّد الأساسي |

### متى تستخدم OpenRouter

**مناسب:**
- تجربة نماذج من مزودين لم تُعدّهم بعد
- اختبار A/B سريع بين عائلات نماذج مختلفة
- فاتورة موحّدة عندما تريد فاتورة API واحدة فقط
- الوصول لنماذج متخصصة أو أحدث غير مدعومة أصليًا بعد

**استخدم التكامل المباشر بدلاً من ذلك عندما:**
- المزوّد مدعوم أصليًا بالفعل (DeepSeek، Anthropic، OpenAI، إلخ)
- تحتاج أقل زمن استجابة ممكن
- تريد ميزات خاصة بالمزوّد (تخزين مؤقت، بادئات الموجّهات، إلخ)
- تُشغّل أحمال عمل إنتاجية عالية الحجم حيث تتراكم القفزة الوسيطة

<Tip>
  إذا كنت تستخدم بالفعل DeepSeek أو Anthropic أو أي مزوّد مدعوم أصليًا، أبقِ على الاتصال المباشر. أضف OpenRouter فقط لنماذج لا يمكنك الوصول إليها مباشرة — ثم اختر نموذج OpenRouter كدماغك حين تريد استخدامه. لا يوجد تدرّج؛ النموذج الذي تختاره هو من يعمل.
</Tip>

## أوضاع الاستدلال

تتحكّم أيقونة **الدماغ** بجوار صندوق الرسائل في طريقة استدلال النموذج. اضغط عليها للتنقّل بين الأوضاع التي يدعمها النموذج المحدّد. تجتمع هنا فكرتان منفصلتان:

### التفكير — *هل* يفكّر النموذج

- **إيقاف** — يجيب النموذج فورًا. الأسرع والأرخص؛ مثاليّ للمهام البسيطة والمباشرة.
- **تشغيل** — يعالج النموذج المسألة أولًا في مرحلة استدلال مخصّصة قبل الردّ. أبطأ ويستهلك رموزًا أكثر، لكنه أدقّ بوضوح في المهام متعدّدة الخطوات أو المنطقية أو الغامضة.

### الجهد — *إلى أي مدى* يفكّر

تُتيح هذا النماذج التي تدعم الجهد فقط، ويُطبَّق عندما يكون التفكير مُفعّلًا.

- **عالٍ** — عمق استدلال قياسي. الخيار الافتراضي المناسب لمعظم المهام.
- **أقصى** — يفكّر النموذج لفترة أطول وأعمق لأصعب المسائل. رموز وكمون أكثر مقابل جودة أعلى في الأعمال المعقّدة.

### حالات الزر

| الحالة | اللون | المعنى |
|-------|------|-------|
| إيقاف | رمادي | التفكير متوقّف — إجابة مباشرة |
| تشغيل | أزرق | التفكير مُفعّل — دون تحكّم بالجهد |
| عالٍ | بنفسجي | التفكير مُفعّل، جهد قياسي |
| أقصى | برتقالي | التفكير مُفعّل، أقصى جهد |

يعرض كل نموذج الحالات التي يدعمها فعليًا فقط. إذا كان النموذج يفكّر دائمًا (لا يمكن إيقافه) أو لا يملك تحكّمًا بالجهد، فإن الزر يعكس ذلك ويُقفَل حيث لا يوجد ما يتغيّر. يحفظ Wolffish اختيارك لكل نموذج.

**في OpenRouter:** يعتمد الاستدلال على النموذج الموجَّه. تعرض النماذج القادرة على الاستدلال إيقاف / عالٍ؛ ولا تملك النماذج غير الاستدلالية تحكّمًا. يحدّ OpenRouter الجهد عند «عالٍ»، وبعض النقاط (مثل GPT-5 وDeepSeek-R) تفكّر إجباريًا — وفيها يتحوّل «إيقاف» إلى حدّ أدنى من الاستدلال.

## استخدام OpenRouter كنموذجك

لا يوجد تدرّج مزودين ولا ترتيب أولوية. لاستخدام نموذج OpenRouter، اختره كدماغك في **الإعدادات ← الأوضاع** (أو كنموذج عامل في [وضع التنسيق](/ar/configuration/orchestrator-mode)). النموذج الذي تختاره هو من يعمل — وليس OpenRouter احتياطيًا تلقائيًا خلف مزوديك الآخرين.
