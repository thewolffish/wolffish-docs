# مهام مستقلة على مؤقّت

ملف النبض يُحدّد المهام المجدولة التي يُشغِّلها وولف فيش بشكل مستقل. يقع في `brain/brainstem/heartbeat.md`، وتُحلّله وحدة brainstem عند بدء التشغيل وتُسجّله كمهام cron متكررة.

## كيف يعمل

كل عنوان `##` في `heartbeat.md` يُحدّد مهمة. نص العنوان هو اسم المهمة، ونص المحتوى هو التعليمة التي تُرسَل إلى الوكيل كرسالة مستخدم في الوقت المجدول.

```markdown
## Morning Summary | Daily (08:00)

Summarize my unread emails and calendar events for today.
List any GitHub notifications. Keep it brief.

## PR Monitor | Every (30m)

Check my open pull requests on GitHub. If any have new
reviews or comments, give me a one-line summary of each.
```

يُحلّل brainstem الجدول من العنوان (بعد فاصل `|`) ويُسجِّل المهمة. عندما يحين وقت الجدول، يُرسَل نص المحتوى إلى الوكيل كما لو كتبه المستخدم.

## صيغ الجدولة

| الصيغة | المعنى | مثال |
|---|---|---|
| `Startup` | يعمل مرة واحدة عند تهيئة brainstem | `Startup` |
| `Every (Nm)` | تكرار بفاصل دقائق | `Every (5m)` |
| `Every (Nh)` | تكرار بفاصل ساعات | `Every (2h)` |
| `Hourly (MM)` | مرة كل ساعة عند الدقيقة المحددة | `Hourly (30)` |
| `Daily (HH:MM)` | مرة يومياً | `Daily (09:00)` |
| `Nightly (HH:MM)` | مرة يومياً (اسم بديل لـ Daily) | `Nightly (23:00)` |
| `Weekday (HH:MM)` | من الاثنين إلى الجمعة فقط | `Weekday (08:30)` |
| `Weekly (Day HH:MM)` | مرة أسبوعياً في يوم محدد | `Weekly (Monday 09:00)` |
| `Monthly (DD HH:MM)` | مرة شهرياً في يوم محدد | `Monthly (01 10:00)` |
| `Cron (expression)` | صيغة cron خام بخمسة حقول | `Cron (*/15 * * * *)` |

<Info>
  جميع الأوقات تستخدم المنطقة الزمنية المحلية لنظامك بتنسيق 24 ساعة. لا يوجد تحويل إلى UTC — `Daily (09:00)` تعني الساعة 9 صباحاً أينما كنت.
</Info>

## تنفيذ المهام

مهام النبض تعمل عبر خط الأنابيب الكامل للوكيل — نفس الدماغ، نفس القدرات، نفس الذاكرة. الفرق الوحيد عن المحادثة العادية:

- **موافقة تلقائية**: استدعاءات الأدوات تتجاوز تأكيد amygdala (بدون نوافذ موافقة)
- **محادثة مغلقة**: كل تشغيل مهمة يُنشئ محادثته الخاصة، مرئية في السجل
- **بدون بث**: تُولَّد الاستجابات في الخلفية بدون مقاطعة واجهة المستخدم

<Warning>
  لأن مهام النبض توافق تلقائياً على استدعاءات الأدوات، كن حذراً في التعليمات التي تعطيها لها. مهمة تقول "احذف الملفات القديمة" ستُنفَّذ بدون سؤال. اكتب تعليمات دفاعية: "اعرض الملفات الأقدم من 30 يوماً وأخبرني بما ستحذفه" أأمن من "احذف الملفات الأقدم من 30 يوماً".
</Warning>

## تفعيل وتعطيل المهام

المهام تكون نشطة عندما يكون عنوانها ومحتواها غير مُعلَّقين. لتعطيل مهمة، علّقها بتعليقات HTML:

```markdown
## Morning Summary | Daily (08:00)

Summarize my unread emails and calendar events for today.

<!--
## Old Job I Don't Need | Every (1h)

This job is disabled and won't run.
-->
```

<Tip>
  يُعيد brainstem قراءة `heartbeat.md` عند بدء التشغيل. بعد تعديل الملف، أعد تشغيل وولف فيش أو انتظر دورة تهيئة brainstem التالية.
</Tip>

## أمثلة عملية

### إحاطة الصباح

```markdown
## Morning Briefing | Weekday (07:45)

Give me a morning briefing:
1. Unread emails — summarize the important ones
2. Today's calendar — list meetings with times
3. GitHub — any PRs that need my review
4. Weather — current conditions and forecast

Keep each section to 2-3 lines max.
```

### فرز البريد الإلكتروني

```markdown
## Email Triage | Every (2h)

Check my email for anything urgent or time-sensitive.
If you find something that needs immediate attention,
flag it and write a one-sentence summary.
Ignore newsletters and marketing emails.
```

### مراقبة طلبات السحب

```markdown
## PR Watch | Every (30m)

Check my open pull requests on GitHub.
Report only PRs with new activity since your last check:
- New reviews or approvals
- CI status changes (especially failures)
- New comments or requested changes

If nothing changed, respond with "No PR updates."
```

### ملخص نهاية اليوم

```markdown
## Daily Wrap-Up | Nightly (18:00)

Write a brief end-of-day summary:
- What tasks were completed today (check conversation history)
- Any pending items that need follow-up tomorrow
- Save this as today's work log in knowledge

Keep it concise — bullet points only.
```

## ضغط الذاكرة

ضغط الذاكرة (تجميع hippocampus) هو عملية مجدولة منفصلة تُعدّ في **الإعدادات > الضغط**، وليس في `heartbeat.md`. يُشغِّل brainstem الضغط وفق جدوله الليلي الخاص، مستخدماً نموذج سحابي لجودة تلخيص أفضل.

<Info>
  الضغط يُجمِّع الأحداث اليومية في ملخصات أسبوعية ويُرقّي المعلومات المهمة إلى ملفات المعرفة. يعمل بشكل مستقل عن مهام النبض.
</Info>

## مثال كامل للملف

```markdown
# Heartbeat

Scheduled jobs for autonomous operation.

## System Check | Startup

Run a quick health check. Verify that all configured
providers are reachable and report any issues.

## Morning Briefing | Weekday (07:45)

Give me a morning briefing:
1. Unread emails — summarize the important ones
2. Today's calendar — list meetings with times
3. GitHub — any PRs that need my review

## PR Watch | Every (30m)

Check my open pull requests on GitHub for new activity.
Only report changes since last check.

## Daily Wrap-Up | Nightly (18:00)

Write a brief end-of-day summary of completed tasks
and pending items for tomorrow.
```
