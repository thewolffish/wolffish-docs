# ماذا يخزّن config.json

يقع في `brain/config.json`، ويخزّن هذا الملف إعدادات على مستوى التطبيق. يُقرأ عند بدء التشغيل ويمكن تعديله عبر واجهة الإعدادات أو بتحرير الملف مباشرة.

## حقول التكوين

```json
{
  "mind": {
    "provider": "deepseek",
    "model": "deepseek-v4-pro"
  },
  "providers": {
    "deepseek": {
      "apiKey": "",
      "model": "deepseek-v4-pro"
    },
    "anthropic": {
      "apiKey": "",
      "model": "claude-sonnet-4-20250514"
    },
    "openai": {
      "apiKey": "",
      "model": "gpt-4o"
    },
    "ollama": {
      "host": "http://localhost:11434",
      "model": "gemma3:4b"
    }
  },
  "language": "en"
}
```

## مرجع الحقول

| الحقل | الوصف |
|---|---|
| `mind.provider` | مزود نموذج اللغة النشط: `deepseek` أو `anthropic` أو `openai` أو `ollama` |
| `mind.model` | اسم النموذج النشط |
| `providers.deepseek.apiKey` | مفتاح API لـ DeepSeek |
| `providers.anthropic.apiKey` | مفتاح API لـ Anthropic |
| `providers.openai.apiKey` | مفتاح API لـ OpenAI |
| `providers.ollama.host` | عنوان خادم Ollama (افتراضيًا: `http://localhost:11434`) |
| `language` | لغة الواجهة: `en` (الإنجليزية) أو `ar` (العربية) |

## اختيار النموذج

يعمل وولف فيش على نموذج مُختار صراحةً — لا يوجد احتياطي تلقائي. في الإعدادات ← الأوضاع، تختار نموذج **الدماغ** لديك (المزوّد + النموذج)، وهو النموذج الذي يعمل. النموذج الذي تختاره هو من يجيب؛ وإن فشل، يفشل الدور بدل التوجيه بصمت إلى مزوّد آخر.

اختياريًا، فعّل **وضع التنسيق** لإضافة نموذج **عامل** ثانٍ يمكن للدماغ تفويض العمل المتوازي إليه. راجع [وضع التنسيق](/ar/configuration/orchestrator-mode) لكيفية عمل ذلك. وحتى عندها لا يوجد تدرّج — نموذجا الدماغ والعامل اللذان اخترتهما هما من يعملان.

جميع المزودين اختياريون — تحتاج فقط الواحد (أو الاثنين) الذي تختاره فعلًا. لاستخدام Ollama، اختره نموذج دماغك؛ لا يوجد انتقال تلقائي إلى المحلي.

<Info>
  تُخزَّن مفاتيح API بنص صريح في config.json. هذا متعمَّد — وولف فيش تطبيق يعمل محليًا أولًا ومساحة عملك ملكك. إذا كنت تستخدم git لإدارة إصدارات مساحة عملك، أضف `brain/config.json` إلى `.gitignore`.
</Info>
