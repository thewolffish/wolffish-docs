# القدرات المضمَّنة

تأتي هذه القدرات ضمن مساحة العمل الافتراضية عند إنشاء Wolffish لمجلد `~/.wolffish/workspace/` عند التشغيل الأول. كل قدرة تتواجد في مجلدها الخاص تحت `brain/cerebellum/`.

<Note>
**بدون حواجز اصطناعية.** بُني Wolffish لنماذج أذكى وأكثر قدرة. أزلنا المهل الزمنية المشفرة وحدود النتائج والقيود التعسفية من جميع إضافات القدرات. النموذج يقرر متى يضع مهلة زمنية وكم نتيجة يطلب. الأوامر تعمل حتى تنتهي طبيعياً. الحدود الوحيدة المتبقية هي حماية الذاكرة من النفاد (100MB للمستندات، 500MB للصوت، 100KB مخزن الإخراج) — تحمي الذاكرة، لا تقيّد الوظائف.
</Note>

## النظام الأساسي

<AccordionGroup>
  <Accordion title="shell — تنفيذ أوامر الشل" defaultOpen>
    **النوع:** إضافة | **المحفزات:** run, execute, command, terminal, shell, bash, npm, npx, git, pip, docker, curl

    تغلّف `child_process` في Node.js لتنفيذ أوامر الشل. الأداة المحورية التي تعتمد عليها العديد من القدرات الأخرى.

    **الأدوات:**
    - `shell_exec` — تشغيل أمر مع دعم اختياري لمسار العمل، المهلة الزمنية (افتراضي 120 ثانية، الحد الأدنى 60 ثانية)، ووضع الخلفية

    **الأمان:**
    - **حظر:** `mkfs`، `dd if=` (تدمير على مستوى القرص)
    - **مدمر:** `rm -rf`، `sudo` (يتطلب موافقة)
    - **تأكيد:** تثبيت الحزم، عمليات docker

    وضع الخلفية (`background: true`) يفصل العملية ويعيد رقم PID فورًا — استخدمه لخوادم التطوير والمراقبين والعمليات طويلة التشغيل.
  </Accordion>

  <Accordion title="filesystem — قراءة وكتابة وتعديل الملفات">
    **النوع:** إضافة | **المحفزات:** file, read, write, edit, create, save, open, patch, modify, folder, directory

    تغلّف `fs` في Node.js لعمليات الملفات الدقيقة دون الحاجة للشل.

    **الأدوات:**
    - `file_read` — قراءة محتوى الملف مع نطاق أسطر اختياري
    - `file_write` — إنشاء أو استبدال ملف (يدعم وضع الإلحاق)
    - `file_patch` — البحث والاستبدال لنص حرفي في ملف

    يُفضَّل استخدام هذه الأدوات بدلاً من `shell_exec` لعمليات الملفات — فهي أسرع وأكثر أمانًا وتنتج سجلات تدقيق أنظف.
  </Accordion>

  <Accordion title="package-manager — تثبيت الحزم عبر المنصات">
    **النوع:** إضافة | **المحفزات:** install, package, brew, winget, apt, dependency

    يكتشف ويستخدم مدير الحزم المناسب للنظام: Homebrew (macOS)، winget (Windows)، apt (Debian/Ubuntu)، dnf (Fedora/RHEL).

    **الأدوات:**
    - `pkg_check` — فحص مدير الحزم المتاح
    - `pkg_install_manager` — تثبيت Homebrew على macOS إن لم يكن موجودًا
    - `pkg_install` — تثبيت حزمة مع حل الاسم المناسب لكل منصة

    القدرات الأخرى تُعلن عن اعتماداتها عبر حقل `packages` في SKILL.md — ومدير الحزم يحل الاسم الصحيح لكل منصة.
  </Accordion>

  <Accordion title="node — بيئة تشغيل Node.js">
    **النوع:** إضافة | **المحفزات:** node, npm, npx, nvm, javascript, js, typescript, ts

    يتحقق من تثبيت Node.js ويثبّته عبر مدير الحزم إن لم يكن موجودًا.

    **الأدوات:**
    - `node_check` — التحقق من تثبيت Node.js والحصول على الإصدار
    - `node_install` — تثبيت Node.js عبر مدير حزم النظام

    بعد التثبيت، استخدم `shell_exec` لأوامر `node` و`npm` و`npx`.
  </Accordion>
</AccordionGroup>

## الويب والبحث

<AccordionGroup>
  <Accordion title="web-search — البحث في الويب وقراءة الصفحات">
    **النوع:** إضافة | **المحفزات:** search, google, look up, find online, web, browse, latest, news, documentation

    بحث ويب واسترجاع صفحات للحصول على معلومات حديثة.

    **الأدوات:**
    - `web_search` — البحث في الويب، يُعيد العناوين والمقتطفات والروابط (حتى 10 نتائج)
    - `web_fetch` — جلب النص الكامل لصفحة ويب (حتى 15000 حرف)

    استخدم `web_search` للاكتشاف، ثم `web_fetch` لقراءة نتائج محددة بالتفصيل. إذا تم تكوين مفتاح Brave API في المتغيرات، يفضل الوكيل أداة بحث Brave بدلاً من ذلك.
  </Accordion>

  <Accordion title="browser — أتمتة المتصفح بدون واجهة">
    **النوع:** إضافة | **المحفزات:** browser, web, website, navigate, login, scrape, screenshot, form, click, automate

    يُطلق نسخة Chromium بدون واجهة للمهام التفاعلية — ملء النماذج، النقر على الأزرار، التنقل في تطبيقات الصفحة الواحدة، واستخراج المحتوى المُنشأ بـ JavaScript.

    - `browser_click` — النقر على عنصر

    - `browser_evaluate` — تنفيذ JavaScript في سياق الصفحة
    - `browser_close` — إنهاء الجلسة

    استخدم `web_fetch` للصفحات الثابتة — استخدم المتصفح فقط عندما تحتاج للتفاعل.
  </Accordion>

    **النوع:** إضافة | **المحفزات:** transcribe, speech to text, stt, whisper, audio to text, dictation

    تحويل الكلام إلى نص محليًا باستخدام OpenAI Whisper. يدعم أكثر من 99 لغة مع كشف تلقائي. يعمل بالكامل محليًا — بدون مفاتيح API، بدون سحابة.

    **الأدوات:**
    - `stt_transcribe` — نسخ ملف صوتي من مسار محدد
    - `stt_transcribe_upload` — نسخ ملف تم رفعه في المحادثة الحالية

    النماذج المتاحة: tiny، base (افتراضي)، small، medium، large. النماذج الأكبر أكثر دقة لكن أبطأ.
  </Accordion>

  <Accordion title="text-to-speech — إنشاء مذكرات صوتية">
    **النوع:** إضافة | **المحفزات:** voice, speak, say, audio, read aloud, voice memo, tts

    تحويل النص إلى كلام عصبي لإنشاء ردود منطوقة ومذكرات صوتية.

    **الأدوات:**
    - `voice_generate` — تحويل النص إلى MP3 (يُعيد مسار الملف)
    - `voice_respond` — الرد بالكامل كمذكرة صوتية (الصوت هو الرد)
    - `voice_list` — عرض جميع ملفات المذكرات الصوتية

    الأصوات تشمل en-US-AriaNeural (أنثى)، en-US-GuyNeural (ذكر)، ar-SA-HamedNeural (عربي ذكر)، ar-SA-ZariyahNeural (عربية أنثى). يستخدم edge-tts.
  </Accordion>

## تخصيص القدرات

القدرات المضمَّنة تُدار وتُحدَّث بواسطة Wolffish — لا تعدّلها مباشرة، فتغييراتك ستُكتب فوقها عند التحديث التالي. بدلاً من ذلك:

- **عطّل** قدرة مضمَّنة من الإعدادات إن لم تحتجها
- **انسخها** — انسخ المجلد باسم جديد، ثم عدّل النسخة كما تشاء
- **أنشئ قدرتك الخاصة** من الصفر باتباع نفس البنية

تسري التغييرات على القدرات المخصصة على الرسالة التالية — لا حاجة لإعادة التشغيل.

<Warning>
  القدرات المخصصة لا تُحفظ احتياطيًا تلقائيًا. إذا حذفت قدرة أنشأتها أنت، فستختفي — ليس لدى Wolffish نسخة منها. استخدم git أو احتفظ بنسخة احتياطية يدوية من مجلد `brain/cerebellum/`. القدرات المضمَّنة تُستعاد تلقائيًا دائمًا عند التشغيل.
</Warning>

<Tip>
  ابدأ بقراءة SKILL.md لأي قدرة مضمَّنة لفهم النمط. ثم أنشئ قدرتك الخاصة باتباع نفس البنية. راجع [إنشاء القدرات](/ar/extending/creating-capabilities) لدليل خطوة بخطوة.
</Tip>
