# Overview

Wolffish searches for today's top AI news, writes a LinkedIn post in your voice, and publishes it to LinkedIn through the browser extension's debugger mode — using your existing Chrome session. No LinkedIn API. No OAuth. No screenshots. Just web search + extension tools dispatching trusted CDP events into your logged-in browser.

## Setup

This workflow is **browser-extension-only** — it never uses computer-use (no screenshots, no pixel clicking, no macOS screen-recording permissions). Read the general [Setting Up for Success](/use-cases/setting-up-for-success) guide for the full picture; what's below is specific to this workflow.

### Required

- **Wolffish installed and running** — the desktop app with a configured brain workspace.
- **Browser extension installed and connected** — this is the whole engine. Open **Settings > Services > Browser Extension**, load the unpacked extension into Chrome or Brave (`chrome://extensions` > Developer Mode > Load Unpacked), and confirm the **status dot is green** before you start. If it isn't connected, the agent has no way to reach LinkedIn and the run dies immediately. See the [Browser Extension docs](/integrations/browser-extension) for the one-time load.
- **Logged into LinkedIn in that browser** — the extension reuses your existing session. Open LinkedIn, confirm you're logged in, and leave the browser open. The agent operates as you — no credentials stored, no login flow.
- **Debugger mode** — the agent attaches Chrome DevTools Protocol (`ext_debugger_attach`) so all interactions (clicks, typing, scrolling) are dispatched as trusted input events instead of detectable DOM calls. LinkedIn's composer is a rich-text editor that rejects programmatic DOM events — debugger mode bypasses this by entering the browser's input pipeline at the same level as physical keyboard and mouse events. Nothing to configure — the agent attaches it itself.
- **Cloud API key** — configured in **Settings > Models**. This is the LLM that powers the agent — the research, curation, writing, and tool orchestration. It does **not** perform web search (that's a separate capability — see Optional below).

### Recommended

- **DeepSeek V4 Pro on Max mode.** Wolffish's recommended default for agentic work, and a strong fit here: frontier-class reasoning and tool-use reliability for the research > curate > post loop, a **1M-token context window** that comfortably holds multi-article reading sessions, and writing sharp enough for a scroll-stopping news roundup. **Max mode** turns on the deep, adaptive reasoning the prompt leans on — selecting newsworthy stories, matching soul.md tone, and the post > verify sequence. At **$0.44 / $0.87 per MTok** it's a fraction of frontier-model cost — which matters because article reading burns tokens fast. Set it per-conversation in Wolffish; see [DeepSeek](/configuration/deepseek). For the cheapest runs drop to DeepSeek V4 Flash; for maximum writing polish, [Claude Sonnet 4.6](/configuration/anthropic) — see the [extension cost table](/integrations/browser-extension#cost-why-model-choice-matters).

### Optional

- **[Brave Search API key](https://brave.com/search/api/)** — configured in Settings > Services > Brave Search. Improves search quality.

### Before You Start

Leave Chrome open with your LinkedIn session active and the extension dot green. The agent uses your existing window — it does not spin up a new browser or a headless session.

## The Prompt

Send this to Wolffish. It researches, writes, and posts — one conversation.

```
ROLE: You are Wolffish in full autonomous agent mode. Work ONLY through the Wolffish
browser extension (ext_* tools) in my real, already-logged-in Chrome tab. Never use
computer-use (computer_screenshot, computer_mouse_click, computer_keyboard_type),
Playwright, browser_* tools, a headless browser, or a new window — those aren't logged
in and LinkedIn returns 403. Don't ask me anything.

MISSION: Research today's top AI news, curate 5-10 stories, write a LinkedIn post in my
voice (match soul.md tone), and publish it — fully autonomous.

═══ CRITICAL RULES ═══
- Extension only: every interaction with LinkedIn goes through ext_* tools. Never take
  a screenshot to orient — use ext_read_page instead.
- Debugger mode FIRST: call ext_debugger_attach before ANY interaction with LinkedIn.
  LinkedIn's rich-text editor rejects programmatic DOM events — CDP input events bypass
  this. Without the debugger attached, typing and clicking WILL fail silently.
- The post is LIVE only when ext_get_url returns a URL that has moved AWAY from the
  composer (no longer contains "shareActive"). A tool's success response is NOT proof.

ENVIRONMENT — extension only:
- Never call computer_screenshot / computer_mouse_click / computer_keyboard_type / any
  computer-use tool.
- Never call browser_launch / browser_navigate / any browser_* (Playwright) tool.
- Don't pass a guessed tabId (tabId:1 fails with "No tab with id"). Omit it to use the
  active tab; if you truly need the id, call ext_tabs_list once and reuse it.

═══ PHASE 1 — RESEARCH ═══

Run multiple web searches:
"AI news today", "LLM breakthroughs this week", "artificial intelligence announcements",
"AI funding rounds", "AI policy regulation". Read the full content of the top articles
using web_fetch.

ONLY use free, open sources that web_fetch can read without hitting a paywall. Good
sources: The Verge, Ars Technica, TechCrunch, arXiv, Papers With Code, Anthropic Blog,
OpenAI Blog, Google AI Blog, Hacker News, r/MachineLearning, VentureBeat, The Register,
CNBC Technology.

DO NOT fetch from paywalled sites — Bloomberg, WSJ, NYT, Financial Times, The
Information. They return HTTP 403 and waste time. If a search result points to a paywalled
domain, skip it and pick the next one. Also skip SEO spam, listicles, AI-generated
summaries of summaries, and aggregator sites that rewrite other outlets with no original
reporting.

═══ PHASE 2 — WRITE ═══

Write a LinkedIn post in my voice (match soul.md tone). Under 3000 characters. Use this
exact format — every line break matters:

🤖 AI News Roundup — N Things You Need to Know Today
(blank line)
1. [topic emoji] [One-sentence headline with full context]
(blank line)
2. [topic emoji] [One-sentence headline with full context]
(blank line)
3. [topic emoji] [One-sentence headline with full context]
(blank line)
... continue for 5-10 stories ...
(blank line)
Generated with Wolffish 🐟

Rules:
- 5 to 10 stories depending on what's newsworthy today
- Each story is exactly one numbered item with a topic emoji
  that matches its domain: 🏦 finance, 🏥 health, 🔬 research,
  💰 funding, 🛡️ policy, 🛒 retail, ☁️ cloud, 🔐 security,
  🎵 consumer, etc.
- Each item is one self-contained sentence — the reader must
  understand the story without clicking through
- One blank line between EVERY item — no exceptions — without
  them LinkedIn renders it as a wall of text
- No hashtags. No closing hot take. No personal reflection.
  The value is the curation.
- End with: Generated with Wolffish 🐟

═══ PHASE 3 — POST (deterministic — follow exactly) ═══

Step 1: Save the post with correct encoding.
Generate a timestamped filename: files/linkedin-post-YYYY-MM-DD-HHMMSS.txt. Write the
post content to that file so emojis are saved with proper UTF-8 encoding.

Step 2: Attach debugger mode.
ext_debugger_attach on the active tab. Wait for confirmation. This routes all subsequent
ext_click / ext_type / ext_keypress calls through CDP — making them indistinguishable from
real keyboard and mouse input. Without this, LinkedIn's rich-text editor will reject
the input.

Step 3: Navigate to the LinkedIn composer.
ext_navigate https://www.linkedin.com/feed/?shareActive=true
ext_wait_for_navigation to confirm the page loaded.

Step 4: Copy post to clipboard and paste.
ext_clipboard_write with the full post text (this handles UTF-8/emoji encoding natively —
no pbcopy workaround needed). Then ext_click the composer text area to focus it, then
ext_keypress "Meta+v" to paste from clipboard.

Step 5: Verify and post.
ext_read_page to confirm the post text is in the composer (check for the "AI News
Roundup" heading and the "Wolffish" footer). If it looks right, ext_click "text=Post" to
submit. Do not take screenshots. Do not scroll through the post.

Step 6: Confirm it went live.
ext_wait 3000 (let LinkedIn process). ext_get_url — if the URL no longer contains
"shareActive", the post was submitted. Done.

Five actions: save file, clipboard-paste, verify, post, confirm. Done.
```

## Understanding the Prompt

The prompt follows the same extension-only pattern as [Engage Reddit](/use-cases/reddit-engage) and [Karma Farm](/use-cases/karma-farm), adapted for LinkedIn's rich-text editor.

| Section | What it does | Why it's there |
|---|---|---|
| **ROLE** | Pins the agent to full-autonomous mode, the extension (not computer-use), your logged-in session. | Stops the agent from falling back to screenshot-based computer-use (slow, expensive, fragile) or launching a headless browser (not logged in). |
| **CRITICAL RULES** | Debugger-first, extension-only, verify with URL check. | LinkedIn's composer is a rich-text editor (like New Reddit's Lexical editor) — it rejects `element.value = '...'` and `element.click()`. CDP events from the debugger bypass this. |
| **PHASE 1 — RESEARCH** | Multi-query web search with source filtering. | Same research pattern as every Wolffish workflow — cast a wide net, skip paywalled/SEO sources, read full articles. |
| **PHASE 2 — WRITE** | Curate, format, enforce blank-line rules. | LinkedIn collapses text without blank lines — the format spec prevents wall-of-text rendering. |
| **PHASE 3 — POST** | Save > debugger > navigate > clipboard-paste > verify > click Post > confirm URL. | The clipboard path (`ext_clipboard_write` + paste) handles emoji encoding natively and works with LinkedIn's contenteditable editor. The URL check confirms the post left the composer. |

<Tip>
  **Why the clipboard path, not ext_type?** LinkedIn's composer is a `contenteditable` div, not a `<textarea>`. Direct typing via `ext_type` works but is slow for 3000-character posts and can drop emojis. `ext_clipboard_write` + paste puts the full text in one shot with correct encoding — the same approach the computer-use version used, but without the screenshot overhead.
</Tip>

## How It Works

1. Prefrontal loads `soul.md` + the browser-extension SKILL.md + web-search SKILL.md into context.
2. The LLM calls `web_search` across multiple queries — news, breakthroughs, funding, policy.
3. The LLM calls `web_fetch` on the top articles to read full content.
4. The LLM curates 5-10 stories and writes the formatted post.
5. The LLM saves the post to a timestamped file for archival.
6. The agent attaches the debugger to the tab (`ext_debugger_attach`) so all input is dispatched as trusted CDP events.
7. `ext_navigate` opens `linkedin.com/feed/?shareActive=true` — the composer loads.
8. `ext_clipboard_write` puts the post text on the clipboard, `ext_click` focuses the composer, `ext_keypress "Meta+v"` pastes.
9. `ext_read_page` confirms the text is in the composer, then `ext_click "text=Post"` publishes.
10. `ext_get_url` confirms the URL moved away from the composer — post is live.

## Limits

- **LinkedIn's composer is a rich-text editor.** Unlike old.reddit's plain `<textarea>`, LinkedIn uses a `contenteditable` div. The clipboard-paste path handles this reliably; direct `ext_set_value` would not work here.
- **Debugger mode is mandatory.** Without `ext_debugger_attach`, LinkedIn's editor ignores programmatic input. The yellow "debugging" banner is visible during the session — if that's a concern, the agent detaches after posting.
- **LinkedIn UI changes are absorbed automatically.** The agent finds elements by text content (`ext_click "text=Post"`), not pixel coordinates — so UI layout changes don't break it the way screenshot-based computer-use would.
- **CAPTCHA / verification walls** can interrupt the run. Debugger mode reduces but does not eliminate this — if LinkedIn challenges the session, the agent will pause and may need you to clear it.
